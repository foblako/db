# Теоретические вопросы для подготовки к зачету

## 1. Что такое SQL? Дать определение, обозначить ключевые моменты
SQL (Structured Query Language) — это язык программирования, предназначенный для управления и манипулирования реляционными базами данных. Ключевые моменты:
- Используется для создания, модификации и управления данными в базах данных.
- Позволяет выполнять запросы для извлечения данных.
- Поддерживает операции вставки, обновления и удаления данных.

## 2. Типы данных SQL. Описать, обозначить особенности
Типы данных в SQL включают:
- **Числовые**: INTEGER, FLOAT, DECIMAL.
- **Символьные**: CHAR, VARCHAR, TEXT.
- **Дата и время**: DATE, TIME, TIMESTAMP.
- **Булевы**: BOOLEAN.
Особенности:
- Каждый тип данных имеет свои ограничения и области применения.
- Выбор типа данных влияет на производительность и объем хранимых данных.

## 3. Классификация команд SQL (5 "языков"). Описать, какие бывают. Описать особенности DDL, DML. Привести примеры
SQL команды делятся на пять категорий:
- **DDL (Data Definition Language)**: команды для определения структуры базы данных (CREATE, ALTER, DROP).
- **DML (Data Manipulation Language)**: команды для манипуляции данными (INSERT, UPDATE, DELETE).
- **DQL (Data Query Language)**: команды для запросов данных (SELECT).
- **DCL (Data Control Language)**: команды для управления доступом к данным (GRANT, REVOKE).
- **TCL (Transaction Control Language)**: команды для управления транзакциями (COMMIT, ROLLBACK).

Примеры:
- DDL: `CREATE TABLE students (id INT, name VARCHAR(100));`
- DML: `INSERT INTO students (id, name) VALUES (1, 'John Doe');`

## 4. Классификация команд SQL (5 "языков"). Описать, какие бывают. Описать особенности DQL, DCL. Привести примеры
SQL команды делятся на пять категорий:
- **DDL (Data Definition Language)**.
- **DML (Data Manipulation Language)**.
- **DQL (Data Query Language)**: команды для запросов данных (SELECT).
- **DCL (Data Control Language)**: команды для управления доступом к данным (GRANT, REVOKE).
- **TCL (Transaction Control Language)**.

Примеры:
- DQL: `SELECT * FROM students;`
- DCL: `GRANT SELECT ON students TO user;`

## 5. Ограничения целостности в SQL
Ограничения целостности обеспечивают корректность данных в базе данных:
- **NOT NULL**: запрещает хранение NULL значений.
- **UNIQUE**: обеспечивает уникальность значений в столбце.
- **PRIMARY KEY**: уникальный идентификатор строки.
- **FOREIGN KEY**: обеспечивает ссылочную целостность между таблицами.
- **CHECK**: проверяет выполнение условия для значений столбца.

## 6. LIMIT/OFFSET. Описание, преимущества/недостатки, практическое применение
- **LIMIT**: ограничивает количество возвращаемых строк.
- **OFFSET**: пропускает указанное количество строк перед началом возврата.
Преимущества:
- Удобны для реализации пагинации.
Недостатки:
- При больших OFFSET может снижаться производительность.
Применение:
```sql
SELECT * FROM students LIMIT 10 OFFSET 20;
```

## 7. Виды пагинации и их особенности
- **LIMIT/OFFSET**: простая реализация, но может быть медленной на больших наборах данных.
- **Keyset Pagination**: использует уникальные ключи для определения границ страниц, более эффективна для больших наборов данных.
- **Cursor-based Pagination**: использует курсоры для определения позиции, обеспечивает стабильную производительность.

## 8. Агрегатные функции: что это такое, какие агрегатные функции поддерживаются в PostgreSQL, особенности использования, FILTER в агрегатных функциях
Агрегатные функции выполняют вычисления над набором значений и возвращают одно значение.
Поддерживаемые функции в PostgreSQL:
- **COUNT()**: количество строк.
- **SUM()**: сумма значений.
- **AVG()**: среднее значение.
- **MIN()**: минимальное значение.
- **MAX()**: максимальное значение.
Особенности:
- Используются с группировкой данных (GROUP BY).
- **FILTER**: позволяет применять агрегатные функции к подмножеству данных.
```sql
SELECT department, COUNT(*) FILTER (WHERE salary > 50000) FROM employees GROUP BY department;
```

## 9. UNION, INTERSECT, EXCEPT: что за операции, практическое применение, особенности, приведение типов
- **UNION**: объединяет результаты двух запросов, исключая дубликаты.
- **INTERSECT**: возвращает общие строки из двух запросов.
- **EXCEPT**: возвращает строки из первого запроса, которых нет во втором.
Применение:
```sql
SELECT name FROM students UNION SELECT name FROM teachers;
```
Особенности:
- Все запросы должны возвращать одинаковое количество столбцов с совместимыми типами данных.

## 10. Декартово произведение: как работает, что это такое, как им пользоваться в Postgres
Декартово произведение (CROSS JOIN) возвращает все возможные комбинации строк из двух таблиц.
Пример:
```sql
SELECT * FROM students CROSS JOIN courses;
```
Особенности:
- Может возвращать большое количество строк, что влияет на производительность.

## 11. INNER JOIN, LEFT JOIN, RIGHT JOIN: как работают, практическое применение, особенности
- **INNER JOIN**: возвращает строки, которые имеют совпадения в обеих таблицах.
- **LEFT JOIN**: возвращает все строки из левой таблицы и совпадающие строки из правой таблицы.
- **RIGHT JOIN**: возвращает все строки из правой таблицы и совпадающие строки из левой таблицы.
Пример:
```sql
SELECT students.name, courses.title FROM students INNER JOIN enrollments ON students.id = enrollments.student_id INNER JOIN courses ON enrollments.course_id = courses.id;
```

## 12. Подзапрос: что это такое, где может быть расположен, особенности, когда используется
Подзапрос — это запрос, вложенный в другой запрос.
Может быть расположен:
- В SELECT, FROM, WHERE, HAVING.
Особенности:
- Может возвращать одно значение, строку или таблицу.
Пример:
```sql
SELECT name FROM students WHERE id IN (SELECT student_id FROM enrollments WHERE course_id = 1);
```

## 13. СТЕ: что это такое, где определяется, особенности, когда и зачем используется
СТЕ (Common Table Expression) — временный результат, определяемый в предложении WITH.
Определяется перед основным запросом.
Особенности:
- Улучшает читаемость и структурированность запросов.
Пример:
```sql
WITH enrolled_students AS (SELECT student_id FROM enrollments WHERE course_id = 1) SELECT name FROM students WHERE id IN (SELECT student_id FROM enrolled_students);
```

## 14. Концептуальная модель данных: процесс разработки, особенности. Сущности, атрибуты
Процесс разработки:
- Определение сущностей и их атрибутов.
- Определение связей между сущностями.
Особенности:
- Абстрактное представление данных.
- Не зависит от СУБД.
Сущности:
- Объекты, представляющие данные (например, студенты, курсы).
Атрибуты:
- Свойства сущностей (например, имя, возраст).

## 15. Концептуальная модель данных: процесс разработки, особенности. Связи, степень связи, мощность связи
Процесс разработки:
- Определение сущностей и их атрибутов.
- Определение связей между сущностями.
Особенности:
- Абстрактное представление данных.
Связи:
- Определяют отношения между сущностями (например, студент записан на курс).
Степень связи:
- Количество сущностей, участвующих в связи (например, бинарная связь между двумя сущностями).
Мощность связи:
- Количество экземпляров одной сущности, связанных с экземплярами другой сущности (например, один ко многим).

## 16. Логическая модель данных: что это такое, на каком этапе разрабатываем, отличия от концептуальной модели
Логическая модель данных:
- Детализированное представление данных, включающее таблицы, столбцы и ключи.
- Разрабатывается после концептуальной модели.
Отличия:
- Логическая модель более детализирована и конкретна.
- Включает технические аспекты реализации.

## 17. Переход от концептуальной модели к логической
Процесс перехода:
- Преобразование сущностей в таблицы.
- Преобразование атрибутов в столбцы.
- Определение первичных и внешних ключей.
- Определение связей между таблицами.

## 18. Нормализация: что это такое, для чего нужна. 1НФ, 2НФ, 3НФ
Нормализация — процесс организации данных для минимизации избыточности и предотвращения аномалий.
1НФ (Первая нормальная форма):
- Удаление повторяющихся групп.
2НФ (Вторая нормальная форма):
- Удаление частичных зависимостей.
3НФ (Третья нормальная форма):
- Удаление транзитивных зависимостей.
